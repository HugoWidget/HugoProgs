## .hps 脚本语法

`.hps`（HugoProgs Script）是 HugoProgs 的批量执行脚本，用于一次性执行多条 HugoProgs 控制台命令，实现自动化操作。脚本由 HugoProgs 主程序解析，可以交互式调用，也可以直接作为命令行参数运行。

### 1\. 基本规则

* **一行一条命令**，不允许在同行书写多条指令。
* **空行**会被忽略。
* **注释**：以 `#` 开头的行视为注释，执行时跳过。**注释不能与命令写在同一行**（即 `#` 必须位于行首，前面不能有有效命令）。
* 每行前后空白字符（空格、制表符）会被自动去除。

### 2\. 命令格式

脚本中的命令格式与 HugoProgs 交互式菜单的输入完全一致：

```
<命令路径> [参数1] [参数2] ...
```

* **命令路径**：必须存在的命令（大小写敏感，按注册时的原样书写）。
* **参数**：命令所需的额外参数，用空格分隔，数量视具体命令而定。

**示例**：  
`mount/mnt 0 1 Z`

* `mount` 是子菜单名，`mnt` 是其中的命令，`0 1 Z` 是三个参数（挂载磁盘0分区1到Z盘）。

`basic/disable`

* `basic` 子菜单，`disable` 命令，无参数。

特别的，在使用common命令可以查看关于变量的相关命令

### 3\. 特殊快捷指令

* **`c<数字>`** ：执行当前菜单下第 N 个命令（按显示顺序）。例如 `c1` 执行第一个命令。
* **`s<数字>`** ：进入当前菜单下第 N 个子菜单。执行后**当前上下文会切换**到该子菜单，后续命令可直接使用其中的命令，除非再用 `..` 返回。
* **`..`** ：返回上一级菜单。可连续使用 `.. ..` 返回多级。

> 注意：数字序号取决于菜单定义顺序，与终端显示一致。但在脚本中建议始终使用**名称路径**以保证可读性和稳定性。

### 4\. 执行上下文

`.hps` 脚本在一个**共享的菜单实例**上逐行执行，因此菜单层级状态会保留，

### 5\. 文件编码

建议文件不含中文以避免此问题

### 6\. 实际示例

一个用于挂载配置盘并查询冰点的脚本：

```
# 提权
privilege

# 挂载配置盘到 X:
mount/conf X

# 查询当前冰点状态
freeze/drv.get

# 完成
pause
```



### 7\. 如何运行脚本

* **命令行直接执行**：  
`HugoProgs.exe C:\scripts\setup.hps`  
（只要文件以 `.hps` 结尾，程序会识别为脚本并执行）
* **在交互式菜单中运行**：  
进入程序后，使用 `hps/run <路径>` 命令。  
示例：`hps/run D:\my.hps`
* **双击运行**：若已通过 `hps assoc` 关联 `.hps` 文件，双击即可执行。

> 脚本中的每条命令执行时都会在终端显示 `[执行] 命令内容`，方便跟踪进度。若某条命令失败，脚本会继续执行下一条（除非命令内部调用 `exit` 退出）。
